# Use Ask User - อย่ากลัวความผิดพลาด

## กฎ

- เมื่อไม่แน่ใจ → ใช้ `ask_user` ถามผู้ใช้
- เมื่อมีหลายทางเลือก → ใช้ `ask_user` ให้ผู้ใช้เลือก
- เมื่อผู้ใช้ย้ำเตือนสิ่งใดๆ → จดจำและถามยืนยันก่อนทำ

## ตัวอย่างสถานการณ์

1. ไม่รู้ว่าผู้ใช้ต้องการ option ไหน → ถาม
2. มีหลายวิธีแก้ปัญหา → ถามว่าอยากได้แบบไหน
3. ผู้ใช้เคยแก้ behavior ของ agent → ยืนยันก่อนทำซ้ำ
4. ไม่มั่นใจว่า path ถูกต้องไหม → ถาม

## ทำไมสำคัญ

- ดีกว่าทำผิดแล้วต้องมาแก้ทีหลัง
- ผู้ใช้มีส่วนร่วมในการตัดสินใจ
- Agent เรียนรู้จาก feedback ของผู้ใช้
- ลดโอกาสทำผิดซ้ำ

## Pattern

```text
1. ระบุปัญหา/ทางเลือก
2. ใช้ ask_user ให้ผู้ใช้เลือก
3. บันทึกการตัดสินใจ
4. ทำตามสิ่งที่ผู้ใช้เลือก
```
